POSSIBLE IMPROVEMENTS — AMES HOUSING PRICES DATASET BY CLAUDE
=====================================================
Last updated: 2026-06-12


1. MISSING VALUE TREATMENT
---------------------------
- The current strategy drops rows with low-missing columns and fills medium-missing
  columns with the mean. Consider alternatives:
    * KNN Imputation or IterativeImputer (MICE) for columns correlated with others,
      which can preserve relationships better than a global mean fill.
    * For columns like GARAGEYRBLT, a missing value likely means "no garage" — encode
      as a separate category or a sentinel value (e.g. 0) rather than imputing a mean.
    * Validate that dropping rows for low-missing columns does not introduce selection
      bias (e.g. if missingness is correlated with price).


2. FEATURE ENGINEERING
-----------------------
- Total area: combine basement, 1st floor, and 2nd floor areas into a single
  TOTALLIVINGSF feature; high individual correlations suggest strong joint signal.
- Age features: derive HOUSE_AGE = YRSOLD - YEARBUILT and REMODEL_AGE = YRSOLD -
  YEARREMODADD. These relative ages are more predictive than raw years.
- Garage capacity proxy: GARAGECARS * GARAGEAREA can capture both size and count.
- Bathroom index: FULLBATH + 0.5*HALFBATH + 0.5*BSMTFULLBATH as a weighted total.
- Has-feature flags: binary columns for HASBASEMENT, HASGARAGE, HASFIREPLACE,
  HASPOOL derived from area/count columns being > 0.
- Porch total: sum OPENPORCHSF + SCREENPORCH + 3SSNPORCH into one TOTALPORCHSF.


3. DROPPED COLUMNS — WORTH RECONSIDERING
------------------------------------------
- NEIGHBORHOOD: dropped due to high cardinality, but it encodes strong location
  signal. Consider target-encoding (mean SALEPRICE per neighborhood) with proper
  cross-validation folds to avoid data leakage.
- CONDITION1/NOISE: dropped after label-encoding showed low correlation. A
  target-encoded version, or a simpler binary NEAR_ROAD flag, might recover signal.
- MOSOLD / YRSOLD: dropped because individual correlations are weak, but a combined
  DATE_SOLD (e.g. YRSOLD + MOSOLD/12) might reveal seasonal or market-cycle trends.
- MSSUBCLASS: currently dropped; could be grouped into broader dwelling types
  (e.g. old/new single-family, townhouse) to reduce cardinality.


4. OUTLIER TREATMENT
---------------------
- The current adaptive IQR strategy removes up to 4% of data per column, applied
  sequentially. This can compound to a large total loss.
    * Apply outlier removal once on the full column list jointly to avoid compounding.
    * For tree-based models (Random Forest, XGBoost), outliers are less harmful —
      consider skipping outlier removal and letting the model handle them.
    * Visualise the SALEPRICE distribution before and after to confirm no price range
      is being systematically excluded.
- SALEPRICE itself is right-skewed. Applying log1p to the target before modelling
  often improves RMSE on linear models and reduces the effect of expensive outliers.


5. TARGET VARIABLE
-------------------
- SALEPRICE is not yet transformed. A log1p transform on the target is standard
  practice for this competition and is expected by most evaluation metrics (RMSLE).
- After modelling, predictions must be back-transformed with expm1.


6. ENCODING STRATEGY
---------------------
- One-hot encoding with drop_first=True can produce multicollinearity with
  features that already have a reference category. Verify that VIF is acceptable
  before linear modelling.
- For tree-based models, label/ordinal encoding of all categoricals (including
  nominal ones) is often sufficient and avoids sparse matrices.
- Consider using pandas CategoricalDtype for ordinal columns — sklearn's
  OrdinalEncoder can then handle unseen categories gracefully at prediction time.


7. CORRELATION-BASED SELECTION LIMITATIONS
-------------------------------------------
- Linear correlation (Pearson) is used throughout as the sole selection criterion.
  This misses non-linear relationships.
    * Supplement with mutual information scores (sklearn.feature_selection.
      mutual_info_regression) for a more complete picture.
    * Use a lightweight tree model (e.g. ExtraTreesRegressor) to get feature
      importances before dropping columns.
- Correlation is computed on the training set only — ensure the same columns
  are dropped for any test/validation set without re-running the selection.


8. DATA LEAKAGE RISKS
----------------------
- Mean imputation in cell [11] uses the global mean, which includes validation rows
  if the full dataset is being processed before a train/test split.
  Fit imputers only on the training fold and transform both folds.
- The outlier removal step filters rows based on the full column distribution —
  this should also be fit only on training data.


9. COLUMN NAMING
-----------------
- '1STFLRSF', '2NDFLRSF', '3SSNPORCH' start with digits, which can break some
  sklearn pipelines and SQL-like operations. Rename to FIRSTFLRSF, SECONDFLRSF,
  THREESSNPORCH (or prefix with COL_).


10. PIPELINE & REPRODUCIBILITY
--------------------------------
- The cleaning is currently a linear sequence of notebook cells with many
  intermediate dataframes. Refactoring into a sklearn Pipeline (or a single
  function) would make it easier to apply identically to the test set and to
  tune inside a cross-validation loop.
- Save the final df_clean to disk (e.g. Parquet) after cleaning so downstream
  modelling notebooks do not need to re-run the entire pipeline.
